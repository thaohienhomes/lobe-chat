AI Provider Management
Kiến trúc quản lý 40+ AI providers

🏗️ Model Runtime Architecture

V2 sử dụng Model Runtime abstraction layer (packages/model-runtime/) — unified LobeRuntimeAI interface cho tất cả providers.

  • Provider as Plugin: Mỗi provider là 1 plugin, dễ thêm mới
  • Factory Pattern: openaiCompatibleFactory cho OpenAI-like APIs
  • Dynamic Routing: Route request đến provider dựa trên user config
  • Zero Vendor Lock-in: Switch provider bất kỳ lúc nào

🔌 40+ Providers Supported

  • Tier 1: OpenAI, Anthropic, Google Vertex AI, Azure
  • Tier 2: AWS Bedrock, Groq, Together AI, Cerebras
  • Tier 3: Straico, NVIDIA NIM, NewAPI, Ollama (local)
  • Speed: Groq, Cerebras — sub-second responses
  • VN Gateway: Vercel AI Gateway (hiện tại Phở Chat dùng)

⚙️ Config & Routing

  • Environment variables: OPENAI_MODEL_LIST bật/tắt models
  • Per-user provider config từ database
  • userId in RouteAttemptResult cho tracking
  • Admin có thể enable/disable providers server-side
  • Users tự thêm API keys cho personal providers

🆚 v1 vs v2 Provider Comparison

v1 (Phở Chat): Vertex AI + Vercel AI Gateway dual provider. Custom routing logic, manual model list.

v2 (Upstream): 40+ providers, automatic routing, factory pattern, community-contributed providers, M2M tokens, Device Gateway.

Upstream Sync Strategy
Chiến lược đồng bộ từ lobehub upstream — tránh lặp lại sai lầm v1

⚠️ Bài học từ v1

Phở Chat v1 fork lobehub rồi diverge 618 commits — sửa trực tiếp code upstream, xen kẽ custom features vào core → không thể merge upstream nữa. V2 phải áp dụng "Thin Fork" strategy từ đầu.

Upstream Sync Flow: upstream/main ──●──●──●──●──●──●──●──●──●── (LobeHub releases v2.2, v2.3...) \ \ pho-chat/main ────────●──○──○──●──────────●──○──○── merge custom merge custom (tag) (isolated) (tag) (isolated) "Thin Fork" Rule: ┌──────────────────────────────────────────┐ │ NEVER modify upstream files directly │ │ ALWAYS add custom code in separate dirs │ │ src/features/pho/ ← custom features │ │ src/plugins/pho/ ← custom plugins │ │ src/config/pho/ ← custom config │ │ api/pho/ ← custom routes │ │ drizzle/pho/ ← custom migrations│ └──────────────────────────────────────────┘

⏰ Tần suất Merge

  • Minor release (v2.x.0): ⭐ Khuyến nghị — mỗi 2-4 tuần
  • Cherry-pick: ⭐⭐ Tốt nhất — chỉ pick commit/PR cần
  • Mỗi tuần: ⚠️ Quá thường xuyên, nhiều noise
  • Major release (v3): 🔴 Rủi ro cao, quá nhiều changes
  • Quy trình: git fetch upstream → git merge upstream/v2.x.0 --no-commit → test → commit

💥 Vùng hay gây Conflict

  • package.json / pnpm-lock: Accept upstream → pnpm install
  • src/store/: Zustand stores → đặt custom tại src/store/pho/
  • src/config/: Model lists → extend thay vì override
  • drizzle/ migrations: DB schema → drizzle/pho/ riêng
  • locales/: i18n keys → Vietnamese overlay, không sửa keys gốc

✅ Checklist mỗi lần Merge

  • 1. git fetch upstream && git log --oneline upstream/main...HEAD
  • 2. Review CHANGELOG upstream → features nào cần?
  • 3. git merge upstream/v2.x.0 --no-commit
  • 4. Resolve conflicts (ưu tiên accept upstream)
  • 5. Chạy pnpm build && pnpm test
  • 6. Chạy ./scripts/rebrand.sh (find-replace branding)
  • 7. Deploy staging → QA → merge to main

🛡️ Nguyên tắc "Thin Fork"

  • Branding: Script rebrand.sh tự động chạy post-merge
  • Features: Tách riêng tại src/features/pho/
  • Config: Dùng PHO_ prefix cho env vars mới
  • CSS: Override bằng CSS custom properties
  • API routes: Thêm tại /api/pho/...

Design System Guide cho Figma / Pencil
Tất cả thông tin cần để thiết kế UI/UX đúng với codebase v2

💡 Tại sao cần Design Brief?

Khi thiết kế UI/UX cho v2 trên Figma/Pencil, design tool cần hiểu: (1) Component library đang dùng, (2) Design tokens (colors, spacing, typography), (3) Screen flows & navigation patterns, (4) Responsive breakpoints. Nếu thiết kế không match với code, developers sẽ mất thời gian tái implement.

🎨 1. Component Library

V2 dùng 3 lớp component:

  • Ant Design 5.x: Base components (Button, Input, Select, Modal, Table, Form, Drawer...)
  • @lobehub/ui: Custom chat components (ChatList, Avatar, Markdown, EditableMessage, PluginStore...)
  • Custom components: Conversation, Portal, Agent panels
  • Icons: @lobehub/icons (AI provider logos, tool icons)

→ Figma: Import Ant Design 5 UI Kit + tạo custom components từ @lobehub/ui

🎨 2. Design Tokens

  • Theming engine: antd-style (CSS-in-JS, token-based)
  • Colors: Ant Design token system — colorPrimary, colorBgContainer, colorText...
  • Dark/Light: Auto theme switching với ThemeProvider
  • Typography: System font stack (-apple-system, BlinkMacSystemFont, Segoe UI)
  • Border Radius: 8px (cards), 12px (modals), 50% (avatars)
  • Spacing: 4px base unit (4, 8, 12, 16, 24, 32, 48)

→ Figma: Tạo Design Tokens library match với antd token names

📏 3. Layout Patterns

  • Desktop: 3-panel layout (Sidebar 280px | Chat | Portal 400px)
  • Mobile: Stack layout, bottom navigation, swipe panels
  • Chat modes: Bubble mode vs Document mode
  • Portal panel: Slide-in right panel (file preview, agent config)
  • Mobile breakpoint: 768px
  • Tablet breakpoint: 1024px

→ Figma: Tạo frames cho Desktop (1440px), Tablet (768px), Mobile (390px)

🤖 Mô tả Logo cho AI Agent tái tạo

Icon Mark "p":
Chữ "p" viết thường (lowercase) theo phong cách calligraphic — vẽ bằng một nét bút liên tục, chảy mượt (single continuous flowing stroke). Nét bút mảnh, đều, có độ dày-mỏng tự nhiên (natural thick-to-thin) như bút brush pen. Chữ p có phần thân dọc (vertical stem) chảy xuống dưới (descender) và phần bụng tròn (bowl) uốn cong thanh lịch. Không có nét trang trí thừa, không có loop phức tạp — chỉ giữ những đường cong essential. Màu Jade Green #059669 trên nền đen #000000.

Wordmark "phở":
Chữ "phở" viết cùng phong cách calligraphic với chữ p icon — các ký tự p, h, ở nối liền nhau thành một dòng chảy liên tục. Chữ "h" kết nối tự nhiên từ chữ "p", chữ "ở" với dấu hỏi (hook above) được vẽ như một nét accent nhỏ gọn phía trên. Toàn bộ wordmark trông như viết bằng tay trong một hơi — elegant, flowing, không có nét lặp hay rườm rà. Cùng màu Jade Green #059669, cùng line weight với icon mark.

Style Keywords: calligraphic, single-stroke, flowing, thin, elegant, minimal, brush-pen, continuous, jade green, black background, luxury monogram
